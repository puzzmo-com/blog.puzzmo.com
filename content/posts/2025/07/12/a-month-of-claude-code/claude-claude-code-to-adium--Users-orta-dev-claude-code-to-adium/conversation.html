Comprehensive Summary: How Adium Message Styles Work

Based on the documentation from /Users/orta/Downloads/CreatingMessageStyles.html, here's a detailed breakdown of how Adium message styles work:

1. File Structure of Adium Message Styles

Adium message styles follow a specific directory hierarchy:

<stylename>.AdiumMessageStyle [directory]
    Contents [directory]
        Info.plist
        Resources [directory]
            Main.css [optional]
            Content.html [optional]
            FileTransferRequest.html [optional]
            Footer.html [optional]
            Header.html [optional]
            Status.html [optional]
            Incoming [directory, optional]
                Content.html [optional]
                NextContent.html [optional]
                Context.html [optional]
                NextContext.html [optional]
                Buddy_icon.png [optional]
                SenderColors.txt [optional]
            Outgoing [directory, optional]
                Content.html [optional]
                NextContent.html [optional]
                Context.html [optional]
                NextContext.html [optional]
                Buddy_icon.png [optional]
            Variants [directory, optional]
                <PrimaryVariant>.css [optional]
                <SecondaryVariant>.css [optional]

2. Template Variables and Keywords

Adium uses a keyword substitution system where special words surrounded by % symbols are replaced with actual content when HTML is rendered. For example:

  • %message% - The actual message content
  • Claude - The sender's name
  • #0000FF - Color assigned to the sender
  • incoming message autoresize - CSS classes for styling different message types
  • %userIcon% - User's buddy icon
  • %topic% - Topic content (for group chats)

These keywords can be used anywhere in HTML files - for content, CSS classes, or any other HTML attributes.

3. HTML Template Files and Their Purposes

Core Template Files:

  • Content.html - Required template for displaying chat messages
  • NextContent.html - Optional alternative template for consecutive messages from the same sender
  • Context.html - Optional template for message history display
  • NextContext.html - Optional template for consecutive messages in history
  • Header.html - Optional template loaded when "Show Header" is enabled
  • Footer.html - Optional template loaded after all other templates (good for JavaScript)
  • Status.html - Optional template for status and event messages
  • Topic.html - Optional template for topic content in group chats
  • FileTransferRequest.html - Optional template for file transfer requests

Template Fallback System:

Adium uses an intelligent fallback system:

  • If Resources/Incoming/NextContent.html isn't found, it uses Resources/Incoming/Content.html
  • If Resources/Outgoing/Content.html isn't found, it uses Resources/Incoming/Content.html
  • If any Context files aren't found, their non-Context equivalents are used
  • If Resources/Incoming/Content.html isn't found, it uses Resources/Content.html

Critical Requirement - id="insert":

Every Status.html and Content.html must contain an element with id="insert" so Adium knows where to place consecutive messages:

<div id="insert"></div>

4. CSS Styling Integration

Main CSS File:

  • Main.css contains primary styling and is usually required unless variants are used
  • Located in Resources/Main.css

Variants System:

  • Variants are alternative CSS files stored in Resources/Variants/
  • Each variant file (e.g., DarkTheme.css) appears as a menu option
  • Variants can import the main CSS: @import "../main.css";
  • If variants exist, they're the only CSS files loaded directly

Special CSS Classes:

  • img.fullSizeImage and img.scaledToFitImage - For image zooming functionality
  • groupchat class - Added to #Chat element for group chat styling
  • Custom classes via incoming message autoresize keyword

5. Technical Implementation Details

Info.plist Configuration:

Required keys:

  • MessageViewVersion - Indicates compatibility (0, 1, 2, 3, or 4)
  • CFBundleName - Style name (< 16 characters, unique identifier)
  • CFBundleIdentifier - UTI string format: <source>.<messagestylename>.<type>

Optional but important keys:

  • DefaultFontFamily & DefaultFontSize - Default text styling
  • ShowsUserIcons - Controls user icon display
  • DisableCombineConsecutive - Prevents consecutive message grouping
  • DefaultBackgroundIsTransparent - Enables transparency
  • AllowTextColors - Controls color display
  • ImageMask - PNG file for masking user icons
  • DefaultVariant - Sets default variant

JavaScript Enhancements:

  • client.debugLog("string") - Logs to Console
  • client.zoomImage(imgElement) - Switches between image classes
  • client.handleFileTransfer("action", "filename") - Handles file transfers

Group Chat Features:

  • groupchat class added to #Chat element
  • Topic editing via <div id="topic"> with contentEditable spans
  • Special %topic% keyword wraps content in editable spans

Color Management:

  • SenderColors.txt - Colon-separated list of CSS-valid colors for #0000FF
  • Per-variant color settings using key:VARIANTNAME format

This system provides a flexible, template-based approach to customizing Adium's chat appearance while maintaining consistency and functionality across different conversation types and user preferences.
